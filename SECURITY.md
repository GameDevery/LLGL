# Security Policy

## Supported Versions

We actively provide security updates for the following versions of LLGL:

| Version | Supported          |
| ------- | ------------------ |
| 0.05    | :white_check_mark: |
| < 0.05  | :x:                |

## Reporting a Vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

If you discover a potential security flaw (e.g., buffer overflows in the Vulkan/D3D12 backends, memory leaks that could lead to DoS, or shader injection risks), please report it follows:

1.  **Email:** Send a detailed report to `llgl-security@lhermanns.me`.
2.  **Details:** Include a proof-of-concept (PoC), the specific backend used (OpenGL, Metal, etc.), and the operating system.
3.  **Response:** You can expect an acknowledgment within 48 hours.

Thank you for helping keep LLGL secure!
